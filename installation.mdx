## System Requirements

Before installing Beils Dashboard, ensure your system meets these requirements:

### Minimum Requirements

<CardGroup cols={2}>
  <Card title="Server Requirements" icon="server">
    - **OS**: Linux (Ubuntu 20.04+), macOS 10.15+, or Windows 10+
    - **RAM**: 2GB minimum, 4GB recommended
    - **Storage**: 5GB free space
    - **CPU**: 2 cores minimum
  </Card>
  <Card title="Software Requirements" icon="code">
    - **Node.js**: v18.0.0 or higher (v20.x recommended)
    - **pnpm**: v8.0.0 or higher
    - **MariaDB/MySQL**: v10.6 or higher
    - **Git**: v2.20 or higher
  </Card>
</CardGroup>

## Prerequisites Installation

<Tabs>
  <Tab title="Ubuntu/Debian">
    ```bash
    # Update package list
    sudo apt update
    
    # Install Node.js 20.x
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
    sudo apt install -y nodejs
    
    # Install pnpm
    npm install -g pnpm
    
    # Install MariaDB
    sudo apt install -y mariadb-server mariadb-client
    
    # Secure MariaDB installation
    sudo mysql_secure_installation
    
    # Install Git
    sudo apt install -y git
    ```
  </Tab>
  
  <Tab title="macOS">
    ```bash
    # Install Homebrew if not already installed
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    
    # Install Node.js
    brew install node@20
    
    # Install pnpm
    npm install -g pnpm
    
    # Install MariaDB
    brew install mariadb
    brew services start mariadb
    
    # Secure MariaDB
    sudo mysql_secure_installation
    ```
  </Tab>
  
  <Tab title="Windows">
    ```powershell
    # Install using Chocolatey package manager
    # First install Chocolatey: https://chocolatey.org/install
    
    # Install Node.js
    choco install nodejs-lts
    
    # Install pnpm
    npm install -g pnpm
    
    # Download and install MariaDB from:
    # https://mariadb.org/download/
    
    # Install Git
    choco install git
    ```
  </Tab>
</Tabs>

<Note>
  Verify installations by running:
  - `node --version` (should be v18+)
  - `pnpm --version`
  - `mysql --version` or `mariadb --version`
  - `git --version`
</Note>

## Development Installation

<Steps>
  <Step title="Clone the Repository">
    Get the source code from your repository:
    
    ```bash
    git clone <your-repository-url> beils-dashboard
    cd beils-dashboard
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:
    
    ```bash
    pnpm install
    ```
    
    This installs the following key dependencies:
    
    <AccordionGroup>
      <Accordion title="Framework & Core">
        - `nuxt@4.3.1` - Full-stack Vue framework
        - `vue@3.5.28` - Progressive JavaScript framework
        - `vue-router@4.6.4` - Official Vue router
        - `@pinia/nuxt@0.11.3` - State management
      </Accordion>
      
      <Accordion title="Database & ORM">
        - `@prisma/client@7.4.1` - Type-safe database client
        - `@prisma/adapter-mariadb@7.4.1` - MariaDB adapter
        - `prisma@7.4.1` - Prisma CLI and schema tools
        - `mariadb@^3.5.1` - MariaDB connector
      </Accordion>
      
      <Accordion title="Authentication & Security">
        - `bcryptjs@^3.0.3` - Password hashing
        - `jsonwebtoken@^9.0.3` - JWT token generation/verification
        - `zod@4.3.6` - Schema validation
      </Accordion>
      
      <Accordion title="UI & Styling">
        - `@tailwindcss/vite@4.1.18` - Tailwind CSS v4
        - `tailwindcss@4.1` - Utility-first CSS framework
        - `daisyui@^5.5.19` - Tailwind component library
        - `lucide-vue-next@0.564.0` - Icon library
      </Accordion>
      
      <Accordion title="Data Fetching & State">
        - `@tanstack/vue-query@5.92.9` - Powerful data synchronization
        - `pinia@3.0.4` - Vue state management
      </Accordion>
      
      <Accordion title="Additional Features">
        - `vue-i18n@11.2.8` - Internationalization
        - `echarts@^6.0.0` - Data visualization charts
        - `vue-echarts@^8.0.1` - Vue wrapper for ECharts
        - `html2pdf.js@^0.14.0` - PDF generation
        - `aos@2.3.4` - Scroll animations
      </Accordion>
    </AccordionGroup>
  </Step>

  <Step title="Database Setup">
    Create a new MariaDB/MySQL database:
    
    ```bash
    # Connect to MariaDB
    sudo mysql -u root -p
    ```
    
    ```sql
    -- Create database
    CREATE DATABASE transpallet_plus_db 
    CHARACTER SET utf8mb4 
    COLLATE utf8mb4_unicode_ci;
    
    -- Create dedicated user (recommended for production)
    CREATE USER 'beils_user'@'localhost' IDENTIFIED BY 'secure_password_here';
    
    -- Grant privileges
    GRANT ALL PRIVILEGES ON transpallet_plus_db.* TO 'beils_user'@'localhost';
    FLUSH PRIVILEGES;
    
    -- Exit
    EXIT;
    ```
    
    <Warning>
      For production, use a strong password and consider limiting privileges to only what's needed (SELECT, INSERT, UPDATE, DELETE).
    </Warning>
  </Step>

  <Step title="Environment Configuration">
    Create a `.env` file in the project root:
    
    ```bash .env
    # ===========================================
    # DATABASE CONFIGURATION
    # ===========================================
    # MariaDB/MySQL connection settings
    DATABASE_HOST=localhost
    DATABASE_USER=beils_user
    DATABASE_PASSWORD=secure_password_here
    DATABASE_NAME=transpallet_plus_db
    DATABASE_PORT=3306
    
    # ===========================================
    # AUTHENTICATION
    # ===========================================
    # JWT secret for token signing (use a long random string)
    JWT_SECRET=your-super-secret-jwt-key-min-32-characters-long
    
    # ===========================================
    # APPLICATION SETTINGS
    # ===========================================
    NODE_ENV=development
    
    # Optional: Custom port for dev server (default 3000)
    # PORT=3000
    
    # Optional: Base URL for production
    # BASE_URL=https://yourdomain.com
    ```
    
    <Tip>
      Generate a secure JWT secret using:
      ```bash
      node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
      ```
    </Tip>
  </Step>

  <Step title="Prisma Setup">
    Initialize Prisma and generate the client:
    
    ```bash
    # Generate Prisma Client based on schema
    pnpm prisma:generate
    ```
    
    The Prisma schema is located at `prisma/schema.prisma` and includes:
    - User management with roles (ADMIN, USER)
    - Client CRM (consents, questionnaires, revokes)
    - Product catalog (brands, categories, subcategories, tags)
    - Services and packs
    - Marketing tools (coupons, bonuses, gift cards)
    - Point of sale (carts, cart items, debts)
    - Appointment booking system
  </Step>

  <Step title="Database Migration">
    Apply the schema to your database:
    
    ```bash
    pnpm prisma:migrate
    ```
    
    This creates all tables, indexes, foreign keys, and constraints defined in the schema.
    
    <Note>
      Migration name will be "init" by default. The migration files are stored in `prisma/migrations/`.
    </Note>
  </Step>

  <Step title="Database Seeding">
    Populate the database with initial data:
    
    ```bash
    pnpm seed
    ```
    
    The seed script (`seeds/seed-db.ts`) creates:
    
    **Users (25 total)**
    - 1 Admin: `admin@beils.com` / `password123`
    - 2 Staff members (ADMIN role)
    - 22 Clients (USER role)
    
    **Brands**
    - Masglo
    - Bioline Jato
    
    **Client Data**
    - 0-3 consent forms per client
    - 0-2 questionnaires per client
    - 0-5 bookings per client (past and future)
    
    <Warning>
      The seed script will delete all existing data before seeding. Only run this in development!
    </Warning>
  </Step>

  <Step title="Start Development Server">
    Launch the application:
    
    ```bash
    pnpm dev
    ```
    
    The server will start at `http://localhost:3000` with:
    - Hot module replacement (HMR)
    - TypeScript type checking
    - Automatic compilation
    - API routes at `/api/*`
    
    Access the dashboard and login with:
    - Email: `admin@beils.com`
    - Password: `password123`
  </Step>
</Steps>

## Production Installation

<Steps>
  <Step title="Environment Setup">
    Create a production `.env` file:
    
    ```bash .env
    # Production database configuration
    DATABASE_HOST=your-db-host.com
    DATABASE_USER=production_user
    DATABASE_PASSWORD=strong-production-password
    DATABASE_NAME=beils_production_db
    DATABASE_PORT=3306
    
    # Strong JWT secret (64+ characters recommended)
    JWT_SECRET=<64-character-random-string>
    
    # Production settings
    NODE_ENV=production
    BASE_URL=https://yourdomain.com
    ```
  </Step>

  <Step title="Build the Application">
    Create an optimized production build:
    
    ```bash
    pnpm build
    ```
    
    This generates:
    - Minified JavaScript bundles
    - Optimized CSS
    - Pre-rendered pages
    - Server-side code in `.output/`
  </Step>

  <Step title="Database Migration">
    Run migrations on production database:
    
    ```bash
    pnpm prisma:migrate
    ```
    
    <Warning>
      Always backup your production database before running migrations!
    </Warning>
  </Step>

  <Step title="Start Production Server">
    Run the production build:
    
    ```bash
    node .output/server/index.mjs
    ```
    
    Or use the preview command for local testing:
    
    ```bash
    pnpm preview
    ```
  </Step>
</Steps>

## Deployment Options

### Option 1: Traditional Server (VPS/Dedicated)

<Steps>
  <Step title="Server Setup">
    ```bash
    # Install Node.js and PM2 process manager
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
    sudo apt install -y nodejs
    npm install -g pm2
    ```
  </Step>

  <Step title="Deploy Application">
    ```bash
    # Clone and build
    git clone <repo> /var/www/beils-dashboard
    cd /var/www/beils-dashboard
    pnpm install
    pnpm build
    
    # Start with PM2
    pm2 start .output/server/index.mjs --name beils-dashboard
    pm2 save
    pm2 startup
    ```
  </Step>

  <Step title="Configure Nginx">
    ```nginx
    server {
        listen 80;
        server_name yourdomain.com;
        
        location / {
            proxy_pass http://localhost:3000;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_cache_bypass $http_upgrade;
        }
    }
    ```
  </Step>
</Steps>

### Option 2: Docker Deployment

Create a `Dockerfile`:

```dockerfile
FROM node:20-alpine

WORKDIR /app

# Copy package files
COPY package.json pnpm-lock.yaml ./

# Install pnpm and dependencies
RUN npm install -g pnpm
RUN pnpm install --frozen-lockfile

# Copy application files
COPY . .

# Generate Prisma client
RUN pnpm prisma:generate

# Build application
RUN pnpm build

EXPOSE 3000

# Start application
CMD ["node", ".output/server/index.mjs"]
```

Create `docker-compose.yml`:

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_HOST=db
      - DATABASE_USER=beils
      - DATABASE_PASSWORD=secure_password
      - DATABASE_NAME=beils_db
      - JWT_SECRET=${JWT_SECRET}
      - NODE_ENV=production
    depends_on:
      - db
  
  db:
    image: mariadb:10.11
    environment:
      - MYSQL_ROOT_PASSWORD=root_password
      - MYSQL_DATABASE=beils_db
      - MYSQL_USER=beils
      - MYSQL_PASSWORD=secure_password
    volumes:
      - db_data:/var/lib/mysql
    ports:
      - "3306:3306"

volumes:
  db_data:
```

Deploy with Docker:

```bash
# Build and start
docker-compose up -d

# Run migrations
docker-compose exec app pnpm prisma:migrate

# View logs
docker-compose logs -f app
```

| Command | Description |

### Backup and Restore

**Backup database:**
```bash
mysqldump -u beils_user -p transpallet_plus_db > backup.sql
```

**Restore database:**
```bash
mysql -u beils_user -p transpallet_plus_db < backup.sql
```

<Tip>
  Schedule regular backups using cron:
  ```bash
  # Daily backup at 2 AM
  0 2 * * * mysqldump -u user -p'password' db > /backups/$(date +\%Y\%m\%d).sql
  ```
</Tip>

## SSL/TLS Configuration

For production, always use HTTPS:

<Steps>
  <Step title="Obtain SSL Certificate">
    ```bash
    # Using Let's Encrypt (free)
    sudo apt install certbot python3-certbot-nginx
    sudo certbot --nginx -d yourdomain.com
    ```
  </Step>

  <Step title="Configure Nginx for HTTPS">
    ```nginx
    server {
        listen 443 ssl http2;
        server_name yourdomain.com;
        
        ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
        
        location / {
            proxy_pass http://localhost:3000;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header Host $host;
        }
    }
    
    # Redirect HTTP to HTTPS
    server {
        listen 80;
        server_name yourdomain.com;
        return 301 https://$server_name$request_uri;
    }
    ```
  </Step>

  <Step title="Auto-renewal">
    ```bash
    # Test renewal
    sudo certbot renew --dry-run
    
    # Certificates auto-renew via cron
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build fails with Prisma errors">
    ```bash
    # Clear Prisma cache and regenerate
    rm -rf node_modules/.prisma
    pnpm prisma:generate
    pnpm build
    ```
  </Accordion>
  
  <Accordion title="Database connection timeout">
    - Check firewall rules allow port 3306
    - Verify database host is reachable: `ping your-db-host`
    - Increase connection timeout in `prisma.config.ts`
    - Check database user permissions
  </Accordion>
  
  <Accordion title="Port already in use">
    ```bash
    # Find and kill process using port 3000
    lsof -ti:3000 | xargs kill -9
    
    # Or use a different port
    PORT=3001 pnpm dev
    ```
  </Accordion>
  
  <Accordion title="JWT token errors">
    - Ensure JWT_SECRET is set in .env
    - Verify token hasn't expired (24h default)
    - Check for special characters in secret that need escaping
    - Clear browser cookies/localStorage and re-login
  </Accordion>
  
  <Accordion title="Memory issues in production">
    ```bash
    # Increase Node.js memory limit
    NODE_OPTIONS="--max-old-space-size=4096" node .output/server/index.mjs
    
    # Or in PM2
    pm2 start .output/server/index.mjs --name beils --node-args="--max-old-space-size=4096"
    ```
  </Accordion>
</AccordionGroup>

## Security Checklist

<Steps>
  <Step title="Environment Variables">
    - [ ] Use strong JWT_SECRET (64+ characters)
    - [ ] Never commit .env to version control
    - [ ] Use different credentials for dev/prod
    - [ ] Rotate secrets periodically
  </Step>

  <Step title="Database Security">
    - [ ] Use strong database passwords
    - [ ] Limit database user privileges
    - [ ] Enable SSL for database connections
    - [ ] Regular backups and backup testing
  </Step>

  <Step title="Application Security">
    - [ ] Enable HTTPS with valid SSL certificate
    - [ ] Keep dependencies updated: `pnpm update`
    - [ ] Configure CORS properly
    - [ ] Implement rate limiting on API routes
    - [ ] Regular security audits: `pnpm audit`
  </Step>

  <Step title="Server Security">
    - [ ] Configure firewall (UFW/iptables)
    - [ ] Disable root SSH login
    - [ ] Set up fail2ban for brute-force protection
    - [ ] Keep OS and packages updated
  </Step>
</Steps>
