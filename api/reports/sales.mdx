## Overview

The sales reports endpoint allows you to retrieve detailed information about sales transactions (carts) in your beauty center. This endpoint supports filtering by status to analyze completed sales, pending transactions, or cancelled orders.

<Info>
This endpoint retrieves data from the `carts` table, which includes all sales transactions along with their items, associated users, and any outstanding debts.
</Info>

## Authentication

This endpoint requires authentication. Include a valid Bearer token in the Authorization header.

```bash
Authorization: Bearer YOUR_TOKEN
```

## Request

### Query Parameters

<ParamField query="status" type="string" optional>
  Filter sales by transaction status. Available values:
  - `pending` - Transactions that haven't been completed
  - `completed` - Successfully completed sales
  - `cancelled` - Cancelled transactions
  
  If not provided, returns all sales regardless of status.
</ParamField>

### Example Requests

<CodeGroup>

```bash Get All Sales
curl --request GET \
  --url https://your-domain.com/api/sales/carts \
  --header 'Authorization: Bearer YOUR_TOKEN'
```

```bash Get Completed Sales Only
curl --request GET \
  --url 'https://your-domain.com/api/sales/carts?status=completed' \
  --header 'Authorization: Bearer YOUR_TOKEN'
```

```javascript JavaScript - Filter by Status
const response = await fetch(
  'https://your-domain.com/api/sales/carts?status=completed',
  {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);

const sales = await response.json();
console.log(sales);
```

```python Python - Get Pending Sales
import requests

url = "https://your-domain.com/api/sales/carts"
params = {"status": "pending"}
headers = {"Authorization": "Bearer YOUR_TOKEN"}

response = requests.get(url, params=params, headers=headers)
sales = response.json()
print(sales)
```

</CodeGroup>

## Filtering and Analytics

### Filter by Status

Use the `status` query parameter to analyze specific transaction types:

- **Completed Sales**: `?status=completed` - Analyze revenue and successful transactions
- **Pending Transactions**: `?status=pending` - Monitor active shopping carts and incomplete sales
- **Cancelled Orders**: `?status=cancelled` - Review cancelled transactions for insights

### Sales Metrics

You can calculate various metrics from the response data:

- **Total Revenue**: Sum of all `total` values for completed sales
- **Average Transaction Value**: Total revenue ÷ number of completed sales
- **Discount Rate**: (Total discounts ÷ Total subtotal) × 100
- **Payment Method Distribution**: Count transactions by `payment_method`
- **Items Sold**: Sum of `quantity` across all cart items
