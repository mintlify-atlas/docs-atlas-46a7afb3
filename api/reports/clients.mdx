## Overview

The client reports endpoint provides comprehensive client data for analytics and reporting. It retrieves all users with the `USER` role (customers/clients) along with aggregated counts of their bookings, consents, and debts. This endpoint supports search functionality to filter clients.

<Info>
This endpoint automatically excludes password information from responses for security purposes.
</Info>

## Authentication

This endpoint requires authentication. Include a valid Bearer token in the Authorization header.

```bash
Authorization: Bearer YOUR_TOKEN
```

## Request

### Query Parameters

<ParamField query="search" type="string" optional>
  Search clients by name, surname, email, or phone number. The search performs a partial match (contains) across these fields.
  
  Example: `?search=maria` will match clients with "Maria" in their name, surname, email, or phone.
</ParamField>

### Example Requests

<CodeGroup>

```bash Get All Clients
curl --request GET \
  --url https://your-domain.com/api/clients \
  --header 'Authorization: Bearer YOUR_TOKEN'
```

```bash Search Clients
curl --request GET \
  --url 'https://your-domain.com/api/clients?search=maria' \
  --header 'Authorization: Bearer YOUR_TOKEN'
```

```javascript JavaScript - Search by Name
const response = await fetch(
  'https://your-domain.com/api/clients?search=garcia',
  {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);

const clients = await response.json();
console.log(clients);
```

```python Python - Search by Email
import requests

url = "https://your-domain.com/api/clients"
params = {"search": "@gmail.com"}
headers = {"Authorization": "Bearer YOUR_TOKEN"}

response = requests.get(url, params=params, headers=headers)
clients = response.json()
print(clients)
```

</CodeGroup>

## Client Analytics

### Key Metrics

Use the response data to calculate important client metrics:

- **Total Active Clients**: Count of clients with `status: "ON"`
- **Client Retention**: Analyze booking frequency and recency
- **Average Bookings per Client**: Calculate from `_count.client_bookings`
- **Clients with Outstanding Debts**: Filter by `_count.debts > 0`
- **Most Engaged Clients**: Sort by `_count.client_bookings` descending

### Search Capabilities

The search parameter supports flexible client lookup:

- **By Name**: `?search=maria` - Finds clients with "Maria" in name or surname
- **By Email**: `?search=@gmail.com` - Finds all Gmail users
- **By Phone**: `?search=612` - Finds clients with matching phone numbers
- **Partial Match**: Search uses "contains" logic for flexible matching

### Relationship Counts

The `_count` object provides insights into client engagement:

- **client_bookings**: Number of appointments scheduled (completed, pending, or cancelled)
- **consents**: Number of signed consent documents (GDPR compliance tracking)
- **debts**: Number of outstanding payment records

## Use Cases

### Client Segmentation
Segment clients based on booking frequency, total visits, or outstanding debts for targeted marketing.

### Customer Lifecycle Analysis
Track client lifecycle from first visit (created_at) through booking history to identify retention patterns.

### Debt Management
Identify clients with outstanding debts for follow-up and payment collection.

### Marketing Campaigns
Use search and filtering to create targeted client lists for promotions and communications.

### Compliance Reporting
Monitor consent document status across your client base for GDPR compliance.

### VIP Client Identification
Identify high-value clients based on booking frequency and engagement metrics.

## Security

<Warning>
The password field is automatically excluded from all client responses for security. Never expose password hashes or tokens in API responses.
</Warning>

- Passwords are removed from the response before sending to clients
- Only users with role `USER` are returned (admins excluded)
- Authentication required for all requests
- Refresh tokens may be included but should be handled securely
