---
title: 'List Bookings'
description: 'Retrieve a list of bookings with optional filtering by date range, staff member, or client'
api: 'GET /api/agenda/bookings'
---

Retrieves a list of all bookings with optional filtering capabilities. Results are ordered by booking date and start time in ascending order.

## Authentication

This endpoint requires authentication via Bearer token in the Authorization header or `auth_token` cookie.

```bash
Authorization: Bearer YOUR_TOKEN
```

## Query Parameters

<ParamField query="start" type="string">
  Filter bookings from this date onwards (ISO 8601 format)
  
  Example: `2024-03-01`
</ParamField>

<ParamField query="end" type="string">
  Filter bookings up to this date (ISO 8601 format)
  
  Example: `2024-03-31`
</ParamField>

<ParamField query="staff_id" type="string">
  Filter bookings assigned to a specific staff member
  
  Must be a valid user UUID (100 characters max)
</ParamField>

<ParamField query="client_id" type="string">
  Filter bookings for a specific client
  
  Must be a valid user UUID (100 characters max)
</ParamField>

## Response

Returns an array of booking objects with related client and staff information.

<ResponseField name="bookings" type="Booking[]">
  Array of booking objects
  
  <Expandable title="Booking Object">
    <ResponseField name="booking_id" type="string">
      Unique identifier for the booking (UUID)
    </ResponseField>
    
    <ResponseField name="client_id" type="string">
      ID of the client who made the booking
    </ResponseField>
    
    <ResponseField name="client" type="object">
      Client information
      <Expandable title="Client">
        <ResponseField name="name" type="string">
          Client's first name
        </ResponseField>
        <ResponseField name="surname" type="string">
          Client's last name
        </ResponseField>
        <ResponseField name="phone" type="string">
          Client's phone number
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="staff_id" type="string | null">
      ID of the assigned staff member (optional)
    </ResponseField>
    
    <ResponseField name="staff" type="object | null">
      Staff member information
      <Expandable title="Staff">
        <ResponseField name="name" type="string">
          Staff member's first name
        </ResponseField>
        <ResponseField name="surname" type="string">
          Staff member's last name
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="item_type" type="string">
      Type of booked item: `service` or `pack`
    </ResponseField>
    
    <ResponseField name="item_id" type="string">
      ID of the service or pack being booked
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Booking status. Possible values:
      - `pending`: Booking created but not confirmed
      - `confirmed`: Booking confirmed
      - `completed`: Service completed
      - `cancelled`: Booking cancelled
      - `no_show`: Client did not show up
    </ResponseField>
    
    <ResponseField name="booking_date" type="string">
      Date of the booking (ISO 8601 format)
    </ResponseField>
    
    <ResponseField name="start_time" type="string">
      Start time in HH:mm format (24-hour)
      
      Example: `09:30`
    </ResponseField>
    
    <ResponseField name="end_time" type="string">
      End time in HH:mm format (24-hour)
      
      Example: `10:30`
    </ResponseField>
    
    <ResponseField name="duration" type="integer">
      Duration in minutes
      
      Example: `60`
    </ResponseField>
    
    <ResponseField name="notes" type="string | null">
      Optional notes or special instructions
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Timestamp when the booking was created
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      Timestamp when the booking was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl -X GET 'https://api.beils.com/api/agenda/bookings' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```bash Filter by Date Range
curl -X GET 'https://api.beils.com/api/agenda/bookings?start=2024-03-01&end=2024-03-31' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```bash Filter by Staff Member
curl -X GET 'https://api.beils.com/api/agenda/bookings?staff_id=abc-123-def-456' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```bash Filter by Client
curl -X GET 'https://api.beils.com/api/agenda/bookings?client_id=xyz-789-uvw-012' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```javascript JavaScript/TypeScript
const response = await fetch('https://api.beils.com/api/agenda/bookings?start=2024-03-01&end=2024-03-31', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  }
});

const bookings = await response.json();
console.log(bookings);
```

```python Python
import requests

response = requests.get(
    'https://api.beils.com/api/agenda/bookings',
    headers={'Authorization': 'Bearer YOUR_TOKEN'},
    params={
        'start': '2024-03-01',
        'end': '2024-03-31'
    }
)

bookings = response.json()
print(bookings)
```

## Notes

- When filtering by date range, both `start` and `end` parameters must be provided together
- Dates are stored in UTC timezone
- Results are automatically ordered by `booking_date` ascending, then `start_time` ascending
- The endpoint includes related client and staff information to minimize additional API calls
