---
title: 'Create Booking'
description: 'Create a new booking for a service or pack'
api: 'POST /api/agenda/bookings'
---

Creates a new booking in the system. The endpoint automatically calculates the `end_time` based on the provided `start_time` and `duration` if not explicitly provided.

## Authentication

This endpoint requires authentication via Bearer token in the Authorization header or `auth_token` cookie.

```bash
Authorization: Bearer YOUR_TOKEN
```

## Request Body

<ParamField body="client_id" type="string" required>
  ID of the client making the booking
  
  Must be a valid user UUID (100 characters max)
</ParamField>

<ParamField body="staff_id" type="string">
  ID of the staff member assigned to this booking
  
  Optional - can be assigned later. Must be a valid user UUID if provided.
</ParamField>

<ParamField body="item_type" type="string" required>
  Type of item being booked
  
  Allowed values: `service`, `pack`
</ParamField>

<ParamField body="item_id" type="string" required>
  ID of the service or pack being booked
  
  Must be a valid service or pack UUID (100 characters max)
</ParamField>

<ParamField body="status" type="string" default="pending">
  Initial status of the booking
  
  Allowed values:
  - `pending`: Default status for new bookings
  - `confirmed`: Pre-confirmed booking
  - `completed`: Mark as completed
  - `cancelled`: Create as cancelled
  - `no_show`: Mark as no-show
</ParamField>

<ParamField body="booking_date" type="string" required>
  Date of the booking
  
  Format: ISO 8601 date string (e.g., `2024-03-15` or `2024-03-15T00:00:00.000Z`)
</ParamField>

<ParamField body="start_time" type="string" required>
  Start time of the booking
  
  Format: HH:mm (24-hour format)
  
  Example: `09:30`, `14:00`
</ParamField>

<ParamField body="end_time" type="string">
  End time of the booking
  
  Format: HH:mm (24-hour format)
  
  If not provided, automatically calculated from `start_time` + `duration`
</ParamField>

<ParamField body="duration" type="integer" default={60}>
  Duration of the booking in minutes
  
  Used to calculate `end_time` if not explicitly provided
  
  Example: `60` for 1 hour, `90` for 1.5 hours
</ParamField>

<ParamField body="notes" type="string">
  Optional notes or special instructions for the booking
  
  Supports text format, max length based on database TEXT type
</ParamField>

## Notes

- If `end_time` is not provided, it's automatically calculated: `end_time = start_time + duration`
- The calculation handles time overflow correctly (e.g., `23:30` + 60 minutes = `00:30`)
- Default duration is 60 minutes if not specified
- Default status is `pending` if not specified
- The `staff_id` field is optional and can be assigned later via the update endpoint
- In production, the service/pack duration could be automatically fetched from the item record instead of requiring manual input
