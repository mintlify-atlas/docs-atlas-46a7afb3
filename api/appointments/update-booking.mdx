---
title: 'Update Booking'
description: 'Update an existing booking by ID'
api: 'PUT /api/agenda/bookings/{id}'
---

Updates an existing booking with new information. The endpoint automatically recalculates `end_time` if `start_time` or `duration` are updated.

## Authentication

This endpoint requires authentication via Bearer token in the Authorization header or `auth_token` cookie.

```bash
Authorization: Bearer YOUR_TOKEN
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier (UUID) of the booking to update
  
  Example: `abc123-def456-ghi789`
</ParamField>

## Request Body

All fields are optional. Only include the fields you want to update.

<ParamField body="client_id" type="string">
  ID of the client
  
  Must be a valid user UUID (100 characters max)
</ParamField>

<ParamField body="staff_id" type="string">
  ID of the staff member assigned to this booking
  
  Set to `null` to unassign staff. Must be a valid user UUID if provided.
</ParamField>

<ParamField body="item_type" type="string">
  Type of item being booked
  
  Allowed values: `service`, `pack`
</ParamField>

<ParamField body="item_id" type="string">
  ID of the service or pack being booked
  
  Must be a valid service or pack UUID (100 characters max)
</ParamField>

<ParamField body="status" type="string">
  Update the booking status
  
  Allowed values:
  - `pending`: Awaiting confirmation
  - `confirmed`: Booking confirmed
  - `completed`: Service completed
  - `cancelled`: Booking cancelled
  - `no_show`: Client did not show up
</ParamField>

<ParamField body="booking_date" type="string">
  Reschedule to a different date
  
  Format: ISO 8601 date string (e.g., `2024-03-15` or `2024-03-15T00:00:00.000Z`)
</ParamField>

<ParamField body="start_time" type="string">
  Update the start time
  
  Format: HH:mm (24-hour format)
  
  Example: `09:30`, `14:00`
  
  Note: Updating this will trigger automatic `end_time` recalculation if `duration` is present
</ParamField>

<ParamField body="end_time" type="string">
  Explicitly set the end time
  
  Format: HH:mm (24-hour format)
  
  If `start_time` and `duration` are both updated, this will be automatically recalculated unless explicitly provided
</ParamField>

<ParamField body="duration" type="integer">
  Update the duration in minutes
  
  Triggers automatic `end_time` recalculation if `start_time` is also present
  
  Example: `60` for 1 hour, `90` for 1.5 hours
</ParamField>

<ParamField body="notes" type="string">
  Update notes or special instructions
  
  Supports text format, max length based on database TEXT type
</ParamField>

```bash cURL - Update Status
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "status": "confirmed"
  }'
```

```bash Assign Staff Member
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \

    "staff_id": "staff-uuid-789"
  }'
```

```bash Reschedule Booking
curl -X PUT 'https://api.beils.com/api/agenda/bookings/abc123-def456-ghi789' \
```

## Notes

- The following fields are protected and cannot be updated: `booking_id`, `created_at`, `updated_at`
- When updating `start_time` and `duration` together, the `end_time` is automatically recalculated
- The recalculation correctly handles time overflow across midnight
- If `booking_date` is provided as a string, it's automatically converted to a proper Date object
- Partial updates are supported - only send the fields you want to change
- The `updated_at` timestamp is automatically updated on every successful update
- To unassign a staff member, explicitly set `staff_id` to `null` in the request
